# Godel Weekly Office Hours Format

**Version:** 1.0  
**Schedule:** Weekly, alternating time zones  
**Duration:** 60 minutes  
**Last Updated:** 2026-02-06

---

## Overview

Weekly office hours provide direct access to the Godel team for questions, demonstrations, and community discussion. Sessions alternate between time zones to serve our global community.

- Complex questions: 3-5 minutes
- Deep discussions: Note for follow-up

## Host Guidelines

### Host Responsibilities

**Before Session:**
- [ ] Review pre-submitted questions
- [ ] Prepare feature spotlight (if applicable)
- [ ] Test audio/video
- [ ] Open relevant documentation links
- [ ] Post reminder in Discord/forum

**During Session:**
- [ ] Start recording
- [ ] Monitor multiple channels (Discord, Forum)
- [ ] Time management
- [ ] Moderate discussions
- [ ] Note follow-up items

**After Session:**
- [ ] Stop and save recording
- [ ] Upload to YouTube (within 24h)
- [ ] Post summary to forum
- [ ] Follow up on unanswered questions
- [ ] Update FAQ with new questions

### Hosting Rotation

| Week | Primary Host | Technical Lead | Community Manager |
|------|--------------|----------------|-------------------|
| 1 | DevRel Lead | Senior Engineer | Community Manager |
| 2 | Product Manager | Platform Engineer | Community Manager |
| 3 | DevRel Lead | Senior Engineer | Community Manager |
| 4 | CTO/Founder | Architect | Community Manager |

### Host Best Practices

**Do:**
- Be authentic and approachable
- Admit when you don't know something
- Connect users with resources
- Encourage community members to answer
- Keep energy positive

**Don't:**
- Go over time on single questions
- Dismiss questions as "already answered"
- Promise features without checking roadmap
- Dominate the conversation
- Rush through to finish early

## Question Management

### Pre-Submission Form

**URL:** https://godel.dev/office-hours

**Fields:**
- Name (optional)
- Question category (dropdown)
- Question text
- Related to (optional context)
- Would you like to screenshare? (yes/no)

**Deadline:** 24 hours before session

### Question Categories

| Category | Typical Response Time | Escalation Path |
|----------|----------------------|-----------------|
| How-to | Immediate | Point to docs |
| Bug Report | Immediate | Create issue |
| Feature Request | Note for roadmap | Tag PM |
| Architecture | Discussion | Schedule follow-up |
| Integration | 5-10 min | Demo if needed |

### Live Question Queue

**Tools:**
- Discord Stage: Built-in raise hand
- Zoom: Raise hand feature
- Google Meet: Hand raise

**Moderation:**
- Community Manager manages queue
- Note order on shared doc
- Call on users by name
- Respect time for each question

## Recording & Distribution

### Recording Process

1. **Pre-Session:**
   - Test recording setup
   - Announce recording to attendees
   - Get consent (implied by staying)

2. **Post-Session:**
   - Export recording within 2 hours
   - Basic editing (trim start/end)
   - Generate transcript (automated)

3. **Publishing:**
   - Upload to YouTube (24h SLA)
   - Create forum post with summary
   - Add to documentation
   - Share on social media

### Video Structure

```
0:00 - Intro/Welcome
0:05 - Feature Spotlight [if applicable]
0:15 - Q&A begins
[Timestamps for each question in description]
50:00 - Closing
```
